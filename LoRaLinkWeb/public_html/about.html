What's this?



You just connected to an open WiFi-Network and found this?

You found a small device based on a microcontroller, which provides communication over radio frequencies. It uses a long range modem, called "LoRa". This modem is capable to transfer small amounts of data over a long distance. The device itself needs only a small amount of energy and is maybe solar and or battery powered. Since the resources on these microcontrollers are limited, it can only serve a small amount of clients at the same time - but you can always build your own node for a few bucks, you only need an ESP32 with a 433MHz LoRa Modem and a SD card. They ESPs are already assembled, so the only difficulty is, to get the firmware on the device and hook up the SD card...

The device supports two types of messages:

ShoutOut and Chat:

A shoutOut is a very short Message(up to 150 chars, depending on encoding), which is "fast" (when it travels across the network, it can and will take some time). A ShoutOut can be read by everyone, who is connected to a Node. This device itself can store a configured amount of messages, if there are too many shoutout messages, the oldest gets deleted... A ShoutOut cannot be encrypted.

A chat message is a personal message to a secific user on a specific device, like an email. A chat can be up to 1500 characters long (depends on encoding and if encryption is used). You can send a chat message by use the "New Chat"-Menu and type a username like <user>@<node> to send a message.

After you send the first message, you can enable end-to-end encryption for the following messages. The encryption happens on the client and is a simple XOR encryption. Without the knwolege of the messages content it's very hard to figure it out. The message is also encryped on the device, but be careful when sending private data, you can never be safe!

What's stored on the device?

During the registration, you have to select a user name and a password and if you want, you can provide an email address to contact you, if the owner of the device decides to, only the owner/admin can see the address. The device can't send any messages via email... Your password is stored as checksum, if you forgot it, you can ask the admin to reset it, if he provided contact data in the owner section, if not, don't forget your password...

How to get involved?



You want to try it out? Just register an account at this device, if it has some free slots...
